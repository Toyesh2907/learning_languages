# C++: A Beautiful, Dangerous Mistake

Welcome to the part of the repository where I willingly decided to learn **C++**.

Yes, I know what you're thinking.

"Why would anyone do this to themselves?"

Good question.

---

## Why I Am Learning C++

Short answer: **because it is gorgeous.**

Long answer:

Modern software is full of "nice" languages that try very hard to protect you from yourself.

Python is a great example.

You write something like:

```python
print("hello world")
```

And everything just works.

How boring.

Meanwhile C++ looks at you and says:

> "Sure. You can print hello world.  
> But first you will understand memory, compilation, linking, headers, templates, and the meaning of suffering."

And honestly?

That’s beautiful.

One must indeed imagine Sisyphus happy

I am Sisyphus and I am shouldering my incompetence of comprehending this language
---

## Python vs C++

Python:
- Friendly
- Comfortable
- Safe
- Lets you live in blissful ignorance

C++:
- Gives you **complete control**
- Gives you **complete responsibility**
- Will happily let you shoot yourself in the foot
- Then ask why you didn't check the pointer first

Some people call this bad design.

I call it **honesty**.

---

## The Segmentation Fault Experience

One of the magical things about C++ is that sometimes your program crashes.

Not with a helpful error.

Not with a traceback.

Just:

```
Segmentation fault (core dumped)
```

You stare at the terminal.

The terminal stares back.

Somewhere deep in your program, you offended the memory gods.

And now you must find out where.

This is called **character development**.

---

## The Real Reason I'm Learning C++

This isn't just academic curiosity.

There are a few practical reasons:

### 1. Machine Learning Reality Check

A lot of machine learning engineers live in Python land.

But here’s the fun part:

Most of the heavy lifting in libraries like TensorFlow, PyTorch, and others is written in **C or C++**.

Python is often just the friendly wrapper.

Which means if you ever want to:

- understand performance
- modify internals
- write custom kernels
- optimize something properly

You eventually run into **C++**.

So learning it now seems like a good investment.

---

### 2. Future Personal Projects

Some projects simply make more sense in C++:

- performance-sensitive systems
- custom tooling
- low-level experimentation
- things where Python would be... questionable

---

### 3. Embedded Engineering

If I ever decide to build something closer to the metal:

- firmware
- embedded systems
- hardware-adjacent programming

C and C++ are still the dominant tools.

So again, learning it now saves pain later.
