# Linguistic Chaos: My Programming Language Journal

Welcome.

This repository exists for one reason:  
I am learning programming languages and I refuse to pretend I know what I'm doing.

Instead of silently suffering through documentation and tutorials like a normal person, I decided to write everything down publicly. This repo is basically my brain dump while I learn things.

Yes, the code will improve over time.  
Yes, some of the early stuff will probably be cursed.

That's the point.

---

## What This Repo Is

- A learning journal for programming languages.
- A place where I document concepts in my own words.
- A log of mistakes, discoveries, confusion, and occasional enlightenment.
- An experiment in learning how to write better technical documentation.

Think of it as lab notes for programming.

---

## What This Repo Is NOT

- Not a polished tutorial.
- Not production-quality code.
- Not a place where everything is correct the first time.

If you find something wrong, congratulations — you discovered the same thing I probably did two days later.

---

## Why This Exists

Two reasons:

1. Writing things down forces understanding.  
   If I can't explain something clearly, I probably don't understand it.

2. Documentation is a skill.  
   And like most skills, you only get better by doing it a lot.

So this repo is basically documentation practice disguised as learning notes.

---

Inside you'll find things like:

- Syntax notes
- Weird language quirks
- Small experiments
- Broken code that taught me something
- Occasional "AHA" moments

---

## Warning to Recruiters

If you are a recruiter who somehow found this repo:

This repository is not meant to showcase polished work.  
It is meant to showcase how I learn.

You will see:

- Rough notes
- Dumb questions
- Gradual improvement
- Occasional profanity directed at compilers, documentation, or my own stupidity

There will also occasionally be cursing, because sometimes the only appropriate response to a compiler error is profanity.

This will be a my 5 stages of grief.

Which, frankly, is what learning engineering looks like in real life.

---

## Ground Rules for This Repo

1. Write everything down.
2. Do not pretend to understand things you don't.
3. Leave the mistakes visible.
4. Improve explanations over time.
5. Future me should be able to read this and go  
   "Oh right, that's how that works."
